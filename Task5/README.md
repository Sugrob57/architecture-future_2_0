## 2. TCO-анализ

### Основные категории затрат

| Категория| Текущая архитектура| Целевая архитектура| 
|----------|--------------------|--------------------|
| Инфраструктура| On-prem (CAPEX)| Cloud (OPEX)| 
| Лицензии| SQL Server, Power BI| Cloud + open-source| 
| Поддержка| Высокая (legacy)| Средняя| 
| Аналитика| Долгая (batch)| Быстрая (streaming)| 
| Разработка| Медленная| Быстрая| 


### Оценка затрат (пример, относительная)

1. В первый год будет дороже (т.к. миграция)
2. С 2-го года — экономия
3. К 3-му году:
    - 20–25% TCO
    - кратное ускорение time-to-market
4. Нематериальные выгоды
    - ускорение time-to-market (x2–x5)
    - Real-time аналитика
    - Масштабируемость
    - Упрощение интеграций


## 3. Roadmap внедрения Data Mesh

### Роли

|Роль   |Ответственность |
|-------|--------------|
|Data Product Owner|владеет витриной данных|
|Data Engineer|пайплайны и стриминг|
|BI Analyst|отчёты|
|Platform Team|инфраструктура|
|Domain Team|бизнес-логика|

### Этапы внедрения

#### Этап 1 (0–6 месяцев) — Пилот

**Цель**: доказать жизнеспособность

1. Выбрать 1–2 домена (Fintech / Patient)
2. Kafka + Schema Registry
3. Один Data Product
4. DLQ + базовый monitoring

#### Этап 2 (6–18 месяцев) — Масштабирование

**Цель**: подключить ключевые домены

1. Подключение ключевых доменов
2. Введение Data Catalog
3. Streaming витрины
4. Anti-Corruption Layer для DWH

#### Этап 3 (18–36 месяцев) — Data Mesh

**Цель**: переход на домены данных

1. Децентрализация
2. Self-service BI
3. Отказ от DWH
4. Event-first интеграции
